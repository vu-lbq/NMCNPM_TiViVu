				Root
				- LICENSE                           : Project license file
				- README.md                         : Project overview and setup instructions
				- requirement.txt                   : Dependencies list (if any)
				- project-structure.txt             : Initial project structure notes
				- apps/                             : Folder chính của Project
					- backend/
						- package.json                  : Backend dependencies, scripts (build/start), metadata
						- pnpm-lock.yaml                : Lock file for exact package versions
						- README.md                     : Backend service documentation
						- server.js                     : Entry point to start Express server and connect DB
						- db/
							- migrations/                 : Sequelize migration scripts to evolve DB schema
							- seed/                       : Seed scripts to populate initial data (e.g., admin user)
						- src/
							- app.js                      : Express app configuration (middleware, CORS, routes)
							- config/                     : Configuration files (DB, env, service keys)
							- controllers/                : Business logic handlers for specific routes
								- controller.js             : Example controller
							- middlewares/                : Auth guard, validation, logging
							- models/                     : Sequelize models and associations
							- routes/
								- mainRouter.js             : Registers API endpoints (auth, chat, admin, voice, vocab)
							- services/                   : External/internal service wrappers (Mailjet, TTS/STT, dictionary)
							- utils/                      : Utility helpers (tokens, formatting, error handling)
						- tests/                        : Backend unit/integration tests
    
						Backend Detailed
						- .env                          : Environment variables (DB URL, JWT, Mailjet)
						- .sequelizerc                  : Sequelize CLI paths (models/migrations/seeders)
						- createdb.js                   : Script to create/init database
						- server.js                     : Express server bootstrap
						- src/
							- app.js                      : Express app setup (middlewares, CORS, routers)
							- dbInit.js                   : Database init/sync logic
							- config/
								- config.json               : Sequelize env configs
								- db.js                     : Sequelize instance connection
							- controllers/
								- adminController.js        : Admin operations (users/roles/dashboard)
								- aiController.js           : AI chat/integration endpoints
								- authController.js         : Register/login/me; token issuance
								- controller.js             : Shared/example controller base
								- conversationController.js : Conversation CRUD/list
								- feedbackController.js     : Feedback submit/list/manage
								- messageController.js      : Message CRUD/list per conversation
								- nlpController.js          : NLP utilities (translate/summarize)
								- passwordController.js     : Forgot/reset password (Mailjet)
								- vocabController.js        : Vocabulary CRUD; phonetics auto-fetch
								- voiceController.js        : TTS/STT/voice-chat endpoints
							- middlewares/
								- auth.js                   : JWT auth guard (ensure user/admin)
								- errorHandler.js           : Error formatting middleware
							- migrations/
								- 20251203000100-create-feedbacks.js           : Create feedbacks table
								- 20251203001000-add-role-to-users.js          : Add role column to users
								- 20251203002000-add-phonetics-to-vocabularies.js : Add phonetics to vocabularies
							- models/
								- Conversation.js           : Conversation model
								- Feedback.js               : Feedback model
								- index.js                  : Models bootstrap/associations
								- Message.js                : Message model
								- User.js                   : User model (email/password/role)
								- Vocabulary.js             : Vocabulary model (word/phonetics)
							- routes/
								- conversationRouter.js     : Conversation/message endpoints
								- mainRouter.js             : Root API router wiring
							- seeders/
								- initall.js                : Seed initial data (admin/users)
							- services/
								- aiService.js              : LLM wrapper (prompting/responses)
								- mailService.js            : Mailjet email sending
								- sttService.js             : Speech-to-text integration
								- ttsService.js             : Text-to-speech integration
							- utils/
								- apiError.js               : Standard API error class
								- apiResponse.js            : Success response helper
								- resetToken.js             : Reset token generation/validation
						- tests/
							- .gitkeep                    : Placeholder for tests folder
					- frontend/
						- index.html                    : Vite HTML shell; mounts React at `#root`
						- package.json                  : Frontend dependencies and Vite scripts
						- pnpm-lock.yaml                : Lock file for frontend packages
						- README.md                     : Frontend app documentation
						- public/
							- images/
								- favicon.svg               : Site icon served from `/images/favicon.svg`
						- src/
							- App.jsx                     : App composition: Router, routes, auth provider wiring
							- main.jsx                    : React/Vite bootstrap: creates root and renders `<App/>`
							- layouts/
								- MainLayout.jsx            : Shared app layout (header/sidebar/content)
							- pages/
								- LandingPage.jsx           : Marketing/entry page; links to sign-in/sign-up
								- LoginPage.jsx             : Sign in/sign up form; Remember Me option; navigation
								- ForgotPasswordPage.jsx    : Request password reset via email flow
								- ResetPasswordPage.jsx     : Reset password using token flow
								- ChatPage.jsx              : Main chat UI; STT/TTS controls; conversation list
								- admin/
									- AdminDashboard.jsx      : Admin overview; quick links/stats
									- AdminUsers.jsx          : Manage users (list, roles); admin-only actions
									- AdminFeedback.jsx       : Review user feedback submissions
							- components/
								- MessageBubble.jsx         : Renders chat messages with styling and actions
								- VoiceChatModal.jsx        : Voice-to-voice modal; record and play AI replies
								- DictionaryModal.jsx       : Word lookup helper; definitions/usage
								- VocabularyModal.jsx       : Saved vocabulary; phonetics + speaker
								- FeedbackModal.jsx         : Collects user feedback; posts to backend
								- Hero.jsx                  : Landing hero section with CTA and branding
								- Carousel.jsx              : Auto-advancing image carousel for landing
								- FeatureTrio.jsx           : Three feature cards highlighting capabilities
							- hooks/
								- useSpeech.jsx             : Browser speech recognition and synthesis helpers
							- context/
								- AuthProvider.jsx          : Auth state, Remember Me, bootstrapping, login/register/logout
								- authContext.js            : React context object for auth state
								- useAuth.js                : Hook to access auth context in components
							- services/
								- api.jsx                   : API client: auth/chat/voice/admin/vocab; attaches JWT
							- utils/
								- sttSanitize.js            : Sanitizes STT output (collapse immediate repeats)
				- docs/
					- architecture_mermaid.mmd        : System architecture diagram (Mermaid)
					- architecture.puml               : System architecture diagram (PlantUML)
					- class-diagram-mermaid.mmd       : Class diagram (Mermaid)
					- class-diagram.puml              : Class diagram (PlantUML)
					- usecase-en.md                   : English use cases description
					- usecase-mermaid.mmd             : Use case diagram (Mermaid)
					- usecase-vi.md                   : Vietnamese use cases description
					- folder_structure.txt            : Project folder structure and file annotations
				- scripts/                          : Helper scripts for development/automation
					- React context object for auth state
